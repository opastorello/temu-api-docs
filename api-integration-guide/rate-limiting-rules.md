# Rate Limiting Rules

To ensure the stable operation of the open platform, we have implemented rate limiting for API requests.Typically, the initial rate limit for each app_key is set to 20 requests per second (qps).

# 

How you can find the limit rules?

You can find information about the rate limiting rules in our platform's documentation.

![image](https://bstatic.kwcdn.com/open-outer/20a3d4b51f/def6f7ab6771fafd1e8a519b078f757a.png)

# 

What should I do if I hit rate limit?

Rate limiting rules are dynamically adjustable. If you urgently need to increase your traffic, please contact us promptly via email at [partner@temu.com](https://mailto:partner@temu.com) or through other available channels. 

We will urgently evaluate the reasonableness of a traffic increase upon receipt of your request and adjust your traffic rules accordingly based on the evaluation results.
