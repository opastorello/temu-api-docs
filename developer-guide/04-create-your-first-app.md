# Create your first App

**Last update:** 2025-01-26 08:36:55

---

This document will detail the steps and necessary explanations for creating an app, an important part of integrating with Temu.

# 
App Type

We currently offer two types of integrations for third-party developers: ERP and WMS. We haven't yet broken down the ERP category into specific functional area directories, but we will be doing so gradually in the future.

- 
Enterprise Resource Planning

Enterprise Resource Planning supports sellers in managing products and shipping orders on it, connects multiple platforms (Temu Shop and other E-commerce platforms) and shares data among the multiple platforms to enable sellers to manage their catalogs and ship their orders more efficiently.

- 
Warehouse Management System

Warehouse Management System app supports sellers in managing warehouse and products stock and package on it, connects multiple platforms (Temu Shop and other E-commerce platforms) and shares data among the multiple platforms to enable sellers to manage their warehouse and product stock and their package more efficiently.

For in-house seller integrations, you only need to select the Self-developed application type of application.

- 
Self-developed application

By self-developed system, manage your shop automatically, simplify product shelving, order processing and data analysis, which helps efficient operations and business growth.

**Note:**

- 
If your target audience is Cross-border sellers, the permissions available for apps may vary depending on the marketplace.

- 
ERP-type apps already include many functional scenarios, covering order management, product management, and inventory management. For any other open requests, please contact: [partner@temu.com](https://mailto:partner@temu.com).

# 
Integration Case

During the application process, there are two related fields to fill in regarding integration cases:

- 
**Integrated Platforms:** Please list the platforms your app has already integrated with. You can input the e-commerce name (Amazon/AliExpress/Lazada/TikTok Shop/Shein/Shopee/Others) or a service market link that you integrated with. You can input as many as you have to accelerate your app creation approval.

- 
**Proof of Integration:** For each platform you have integrated with, you need to provide proof. Generally, this can be a screenshot from the app store.

# 
Document

Finally, you'll need to provide a brief market requirements document and a product requirements document. These two documents have different focuses:

- 
**Product Requirements Document:** This document should focus on describing which APIs the product needs to implement which functional scenarios.

- 
**Market Requirements Document:** This document mainly describes how many sellers are using the function and how it compares to existing solutions. This can be filled out briefly. Generally, innovative features will undergo a stricter review of this document.
